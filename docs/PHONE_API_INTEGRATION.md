# Интеграция API для работы с телефоном

## Обзор

Добавлена поддержка регистрации и авторизации по номеру телефона **только с SMS-подтверждением** (без паролей). Все новые эндпоинты пока **замокированы** и готовы к быстрому размокиванию при запуске новой версии сервера.

## Новые эндпоинты

### 1. Инициализация регистрации по телефону
**Эндпоинт:** `POST /api/register/phone/init`

**Запрос:**
```json
{
  "phone": "+79123456789"
}
```

**Успешный ответ (200):**
```json
{
  "success": true,
  "message": "SMS код отправлен на указанный номер",
  "phone": "+79123456789",
  "expires_in": 600
}
```

**Ошибки:**
- 400 - Невалидный номер телефона
- 409 - Номер телефона уже зарегистрирован

### 2. Завершение регистрации по телефону
**Эндпоинт:** `POST /api/register/phone/complete`

**Запрос:**

```json
  "name": "Тестовый Пользователь",
  "city": "Москва",
```

```json
  "token_type": "Bearer",

  "user_id": "7dea49e0-9662-42dc-91cc-729316a411f2",
  "user": {
    "id": "7dea49e0-9662-42dc-91cc-729316a411f2",

    "email": null,
    "phone": "+79987654321",
    "city": "Москва",
    "skill_level": "любитель",
    "avatar_url": "https://ui-avatars.com/api/?name=Тестовый%20Пользователь&background=random",
    "is_club_admin": false,
    "club_id": null,
    "status": "active",
```

**Ошибки:**
- 400 - Невалидные данные

- 422 - Невалидный SMS код

### 3. Инициализация входа по телефону

**Запрос:**
```json
{
  "phone": "+79987654321"
```

**Успешный ответ (200):**

## Пайплайн регистрации по телефону

1. **Инициализация:** `POST /api/register/phone/init`
   - Пользователь вводит номер телефона
   - Сервер отправляет SMS с кодом

2. **Подтверждение:** Пользователь вводит SMS код

3. **Завершение:** `POST /api/register/phone/complete`
   - Пользователь заполняет профиль (имя, город, уровень) - **БЕЗ ПАРОЛЯ**
   - Создается аккаунт и возвращается JWT токен

## Пайплайн авторизации по телефону

1. **Инициализация:** `POST /api/login/phone/init`
   - Пользователь вводит номер телефона
   - Сервер отправляет SMS с кодом

2. **Подтверждение:** `POST /api/login/phone`
   - Пользователь вводит SMS код
   - Возвращается JWT токен

## Измененные файлы

### Модели (`lib/models/user.dart`)
- `PhoneInitRequest` - запрос инициализации SMS
- `PhoneInitResponse` - ответ с подтверждением отправки SMS
- `PhoneCompleteRequest` - запрос завершения регистрации (**БЕЗ ПАРОЛЯ**)
- `PhoneLoginRequest` - запрос авторизации по телефону (**С SMS КОДОМ**)

### API сервис (`lib/services/api_service.dart`)
- `initPhoneRegistration()` - инициализация регистрации по телефону
- `completePhoneRegistration()` - завершение регистрации по телефону (без пароля)
- `initPhoneLogin()` - **НОВЫЙ** - инициализация входа по телефону
- `loginWithPhone()` - авторизация по телефону (с SMS кодом)

### Экраны
- `lib/screens/register_screen.dart` - добавлен переключатель email/телефон
- `lib/screens/phone_verification_screen.dart` - экран ввода SMS кода для регистрации
- `lib/screens/phone_complete_registration_screen.dart` - завершение регистрации по телефону (**БЕЗ ПОЛЕЙ ПАРОЛЯ**)
- `lib/screens/login_screen.dart` - добавлен переключатель email/телефон, убрано поле пароля для телефона
- `lib/screens/phone_login_verification_screen.dart` - **НОВЫЙ** - экран ввода SMS кода для входа

## Моки и тестирование

### Текущие моки:
- **SMS код:** `123456` (для всех операций)
- **Любой номер телефона** принимается для регистрации и входа
- **Пароли не требуются** для телефонной аутентификации

### Как размокать:

1. В `lib/services/api_service.dart` найти методы:
   - `initPhoneRegistration()`
   - `completePhoneRegistration()`
   - `initPhoneLogin()` - **НОВЫЙ**
   - `loginWithPhone()`

2. Закомментировать блоки с моками (помечены `// МОКАЕМ ПОКА ЧТО`)

3. Раскомментировать блоки с реальными HTTP запросами (помечены `// РАЗМОКАТЬ КОГДА СЕРВЕР БУДЕТ ГОТОВ`)

## Особенности новой реализации

- **Без паролей:** Телефонная аутентификация полностью основана на SMS кодах
- **Безопасность:** SMS коды имеют время жизни (600 секунд)
- **UX:** Автоматическое переключение между полями ввода кода
- **Простота:** Меньше полей для заполнения при регистрации
- **Современность:** Соответствует современным стандартам мобильной аутентификации
