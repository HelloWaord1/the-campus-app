# 📅 Интеграция с Apple Calendar

## 🎯 Описание функциональности

Реализована интеграция с системным календарем iOS для добавления тренировок в личный календарь пользователя. При нажатии на кнопку "Добавить в календарь" открывается нативный диалог iOS, позволяющий пользователю выбрать календарь и сохранить событие.

## 📦 Установленные зависимости

### pubspec.yaml
```yaml
dependencies:
  add_2_calendar: ^3.0.1
```

**Плагин `add_2_calendar`:**
- ✅ Простая интеграция
- ✅ Нативный диалог iOS/Android
- ✅ Автоматическое управление разрешениями
- ✅ Поддержка напоминаний

## 🔧 Реализованные компоненты

### 1. **CalendarUtils** (`lib/utils/calendar_utils.dart`)

Утилита для работы с календарем:

**Основные методы:**
- `addTrainingToCalendar(Training training)` - добавление тренировки в календарь
- `formatEventDate(DateTime dateTime)` - форматирование даты события

**Данные события:**
```dart
Event(
  title: training.title,
  description: "Описание + Тренер + Уровень + Цена",
  location: "Клуб, Город, Адрес",
  startDate: training.startTime,
  endDate: training.endTime,
  reminder: Duration(hours: 1), // Напоминание за 1 час до начала
)
```

### 2. **TrainingDetailScreen** (обновлен)

**Добавлено:**
- Импорт `calendar_utils.dart`
- Метод `_addToCalendar()` для обработки добавления в календарь
- `GestureDetector` на кнопке "Добавить в календарь"
- Уведомления об успехе/ошибке через `NotificationUtils`

**Интеграция:**
```dart
GestureDetector(
  onTap: _addToCalendar,
  child: const Text('Добавить в календарь', ...),
)
```

### 3. **iOS Permissions** (`ios/Runner/Info.plist`)

Добавлено разрешение:
```xml
<key>NSCalendarsUsageDescription</key>
<string>Приложению требуется доступ к календарю для добавления тренировок в ваш личный календарь</string>
```

## 📱 Как работает функциональность

1. **Пользователь открывает тренировку** → `TrainingDetailScreen`
2. **Нажимает "Добавить в календарь"** → вызывается `_addToCalendar()`
3. **Система запрашивает разрешение** (первый раз) → показывается системный диалог iOS
4. **Открывается нативный диалог календаря** → пользователь выбирает календарь
5. **Событие добавляется** → показывается уведомление об успехе

## 🎨 UX особенности

### ✅ Что добавляется в календарь:

- **Название события**: название тренировки
- **Время**: начало и окончание тренировки
- **Место**: клуб, город, адрес
- **Описание**:
  ```
  [Описание тренировки]
  
  Тип: Групповая тренировка
  Тренер: Алексей Смирнов
  Уровень: 2.5-3.5
  Стоимость: 1500₽
  Участников: 4/8
  ```
- **Напоминание**: за 1 час до начала

### 🔔 Уведомления:

- ✅ **Успех**: "Тренировка добавлена в календарь"
- ❌ **Ошибка**: "Не удалось добавить тренировку в календарь"

## 🧪 Тестирование

### Сценарии для тестирования:

1. **Первое добавление**:
   - Открыть любую тренировку
   - Нажать "Добавить в календарь"
   - Система должна запросить разрешение
   - Разрешить доступ
   - Должен открыться диалог календаря
   - Сохранить → проверить в приложении Календарь

2. **Повторное добавление**:
   - Разрешение уже дано
   - Сразу открывается диалог календаря

3. **Отказ в разрешении**:
   - При первом запросе отказать
   - Должно показаться сообщение об ошибке

4. **Проверка данных события**:
   - Открыть Календарь iOS
   - Найти добавленное событие
   - Проверить все поля (название, время, место, описание)

### Тестовые данные:

Используйте любую тренировку из списка:
- Групповую (для проверки количества участников)
- Индивидуальную (без участников)
- С разными клубами и тренерами

## 🛠️ Troubleshooting

### Проблема: "Не удается добавить в календарь"

**Решение:**
1. Проверить разрешения в Настройках → Paddle App → Календари
2. Убедиться, что есть хотя бы один календарь в системе
3. Перезапустить приложение

### Проблема: "Диалог не открывается"

**Решение:**
1. Проверить `Info.plist` на наличие `NSCalendarsUsageDescription`
2. Переустановить приложение (удалить и установить заново)
3. Проверить логи: `flutter logs`

### Проблема: "События дублируются"

**Решение:**
- Плагин `add_2_calendar` не проверяет дубли
- Пользователь видит все события в диалоге и может отменить добавление
- Это ожидаемое поведение
